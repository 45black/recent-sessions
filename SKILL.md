# /recent-sessions — Cross-IDE & Cloud Session Surfacing

Lists recently active sessions across every coding environment this user runs — local IDE agents, Codex/Gemini CLIs, VS Code-family workspaces, git repos on mounted volumes, and Anthropic's Managed Agents API — so you can pick up where you left off without hunting through history.

## When to use

- "What was I working on yesterday?" across Claude Code + Cursor + Codex + Anthropic cloud.
- Before starting new work — check whether an older session deserves resuming.
- After a context switch / machine reboot — reconstruct what's mid-flight.
- Triaging handover: see which projects had touches in the last day/week.

Do NOT use for: in-session token/cost tracking (that's `session-management`), or for resuming a specific Claude Code session by ID directly (that's the harness's built-in resume).

## First-run setup

Before anything runs, the user should confirm which sources to enable and how cloud auth is supplied. The skill ships with an interactive setup:

```
bash ~/.claude/skills/recent-sessions/scripts/setup.sh
```

The setup script:

1. Probes every known source and prints whether it was found on this machine.
2. Asks per-source enable toggles.
3. If Anthropic cloud is enabled, asks how to supply the API key — Doppler (cached to a 0600 env file under `~/.claude/secrets/`), an existing env file the user maintains, or `$ANTHROPIC_API_KEY` in the shell at runtime.
4. Writes `~/.claude/skills/recent-sessions/config.env` containing the enable flags and path to the secrets file.

Re-run setup any time to change toggles — it overwrites `config.env`. The file is plain shell and can be hand-edited.

## Architecture

Modular — one file per source under `scripts/sources/`. Adding a new source means adding a `scan_<name>()` function to a new file and registering it in the dispatcher. Nothing else changes.

```
~/.claude/skills/recent-sessions/
├── SKILL.md
├── config.env                        # generated by setup.sh
└── scripts/
    ├── list-recent-sessions.sh       # dispatcher
    ├── setup.sh                      # interactive configurator
    └── sources/
        ├── _common.sh                # emit(), file_mtime(), iso_to_epoch()
        ├── claude.sh
        ├── codex.sh
        ├── gemini.sh
        ├── cursor.sh
        ├── vscode.sh                 # plus _scan_vscode_like() used by cursor.sh
        ├── volumes.sh
        └── anthropic.sh              # Managed Agents cloud
```

Every scanner emits TSV rows via the shared `emit` helper:

```
mtime_epoch \t mtime_iso \t source \t project \t identifier \t resume_hint
```

The dispatcher sorts, limits, and formats. Add a cloud source by dropping `sources/<new>.sh` into the tree, giving it a `scan_<new>()` that calls `emit`, and adding the corresponding `SOURCE_<NEW>` toggle to `config.env` plus a `load_source <new> SOURCE_<NEW>` line in the dispatcher.

## Sources

| Source     | Location                                                          | Identifier used   | Resume hint                   |
|------------|-------------------------------------------------------------------|-------------------|-------------------------------|
| claude     | `~/.claude/projects/<project>/*.jsonl`                            | session UUID      | `claude --resume <id>`        |
| codex      | `~/.codex/{sessions,archived_sessions}/rollout-*.jsonl`           | rollout ID        | `codex (rollout <id>)`        |
| gemini     | `~/.gemini/history/<project>/`                                    | project name      | `gemini (history dir: <p>)`   |
| cursor     | `~/Library/Application Support/Cursor/User/workspaceStorage/*/`   | workspace hash    | `cursor "<folder>"`           |
| vscode     | `~/Library/Application Support/Code/User/workspaceStorage/*/`     | workspace hash    | `code "<folder>"`             |
| volumes    | `/Volumes/*/**/.git` (opt-in; slow)                               | repo path         | `cd "<repo>" && git status`   |
| anthropic  | `GET https://api.anthropic.com/v1/sessions` (Managed Agents beta) | session ID        | `anthropic session <id>`      |

All local sources use file mtime as "last active". Claude Code jsonl is authoritative. Cursor/VS Code `workspaceStorage` mtime reflects when the IDE last touched the workspace's state — a good proxy. The Anthropic scanner reads `updated_at`/`last_used_at`/`created_at` from the API response, in that order.

## How to run

```bash
# Respect config.env toggles:
bash ~/.claude/skills/recent-sessions/scripts/list-recent-sessions.sh --hours 168 --limit 20

# Ad-hoc — ignore config, hit just one source:
bash ~/.claude/skills/recent-sessions/scripts/list-recent-sessions.sh --source anthropic --hours 168

# Multiple sources in one run:
bash ~/.claude/skills/recent-sessions/scripts/list-recent-sessions.sh --source claude --source anthropic

# JSON Lines for piping:
bash ~/.claude/skills/recent-sessions/scripts/list-recent-sessions.sh --json | jq -c

# Run the interactive setup again:
bash ~/.claude/skills/recent-sessions/scripts/list-recent-sessions.sh --setup
```

Flags:

- `--hours N` — window (default 72)
- `--limit N` — max rows (default 30)
- `--source NAME` — restrict to named source; repeat for multiple. Overrides `config.env`.
- `--json` — JSON Lines rather than the default aligned table
- `--setup` — re-enter the interactive configurator and exit

## config.env reference

The setup writes this; it's plain shell so you can hand-edit. Example contents:

```bash
# Source toggles — 1 enabled, 0 disabled.
SOURCE_CLAUDE=1
SOURCE_CODEX=1
SOURCE_GEMINI=1
SOURCE_CURSOR=1
SOURCE_VSCODE=1
SOURCE_VOLUMES=0
SOURCE_ANTHROPIC=1

# Anthropic cloud auth: either a path to an env file that exports
# ANTHROPIC_API_KEY, or leave unset and export it in the shell yourself.
ANTHROPIC_SECRETS_FILE="$HOME/.claude/secrets/recent-sessions-anthropic.env"
ANTHROPIC_BETA_HEADER="managed-agents-2025-09-25"
# ANTHROPIC_API_BASE="https://api.anthropic.com"
# ANTHROPIC_SESSION_LIMIT=100

# Path overrides — only set if a source lives somewhere non-default.
# CLAUDE_PROJECTS_DIR="$HOME/.claude/projects"
# CODEX_SESSION_DIR="$HOME/.codex/sessions"
# VOLUMES_ROOTS="/Volumes/Dev /Volumes/ForgeRuntime"
```

## Invocation pattern for Claude

When invoked, run the dispatcher with `--hours` tuned to the user's phrasing (default 72h; "today" ≈ 24, "this week" ≈ 168, "yesterday" ≈ 48). Summarise the top hits **grouped by source** so the user can scan one tool at a time. For each hit show:

- Relative time + absolute UTC timestamp.
- Source and project path.
- Best identifier to resume it.
- The resume hint verbatim (already formatted by the scanner).

If `config.env` is missing, offer to run `--setup` before producing results — the output will be incomplete without cloud auth, and the user may not have opted into `volumes` scanning yet.

## Anthropic cloud — notes

- Uses the documented Managed Agents beta endpoint `GET /v1/sessions`. The beta header defaults to `managed-agents-2025-09-25`; Anthropic bumps this occasionally, so the setup exposes it as an editable value.
- Reads defensively: `.sessions[]` or `.data[]`, fields `id`/`session_id`, `updated_at`/`last_used_at`/`created_at`, `agent_id`/`agent`/`name`. Response-shape drift between betas won't break the scanner as long as those field families exist.
- On HTTP ≠ 200 the scanner prints a one-line diagnostic to stderr and exits cleanly — other sources still complete.
- Auth precedence at runtime: if `ANTHROPIC_SECRETS_FILE` is set in `config.env` and the file exists, it's sourced. Otherwise the scanner relies on whatever `$ANTHROPIC_API_KEY` the shell provides.
